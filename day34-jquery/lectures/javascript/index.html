Loops and Arrays

Arrays

Arrays are used to store multiple values in a single variable.

var cars = ["Saab", "Volvo", "BMW"];

Using Array

You refer to an array element by referring to the index number.

This statement accesses the value of the first element in cars: 

var name = cars[0];

This statement modifies the first element in cars: 

cars[0] = "Opel";

The real strength of JavaScript arrays are the built-in array properties and methods: 

var x = cars.length;
var y = cars.sort();

Loops

We have declared array with three different car vendors.

Imagine, we want to print this vendors in rows: 

text += cars[0] + "<br>"; 
text += cars[1] + "<br>"; 
text += cars[2] + "<br>";
... it be also written as: 
for (i = 0; i < cars.length; i++) { 
    text += cars[i] + "<br>";
}

Types of loops

  • Loops with unknown number of iterations
    • Loop with condition in the beginning
    • Loop with condition in the end
  • Loops with known number of iterations

Loops with unknown number of iteration

Loop with condition in the beginning

while(condition){
	code_to_execute
}

While loop checkes, if condition is true and run code. If condition is false, loop ends.

Loop with condition in the end

do{
	code_to_execute
}while(condition)

Do-While loop run code and then check the condition. If condition is false, loop ends.

Loops flowchart

Loop with known number of iterations

for - loops through a block of code a number of times

for (statement 1; statement 2; statement 3) {
    code block to be executed
}

Statement 1 is executed before the loop (the code block) starts.
Statement 2 defines the condition for running the loop (the code block).
Statement 3 is executed each time after the loop (the code block) has been executed.

example 

for (i = 0; i < 5; i++) {
    text += "The number is " + i + "<br>";
}

Working with cookies

What are Cookies?

Cookies are data, stored in small text files, on your computer.

When a web server has sent a web page to a browser, the connection is shut down, and the server forgets everything about the user.

Cookies were invented to solve the problem "how to remember information about the user".

Create cookie

document.cookie = "username=John Doe; expires=Thu, 18 Dec 2013 12:00:00 UTC";

<script>
function setCookie(cname, cvalue, exdays) {
    var d = new Date();
    d.setTime(d.getTime() + (exdays*24*60*60*1000));
    var expires = "expires="+ d.toUTCString();
    document.cookie = cname + "=" + cvalue + ";" + expires + ";path=/";
}
</script>

Read cookie

            
<script>
var x = document.cookie;
</script>

document.cookie will return all cookies in one string much like: cookie1=value; cookie2=value; cookie3=value;

<script>
function str_obj(str) {
    str = str.split(', ');
    var result = {};
    for (var i = 0; i < str.length; i++) {
        var cur = str[i].split('=');
        result[cur[0]] = cur[1];
    }
    return result;
}
</script>

Change cookie

Change cookie the same way as create - old one, will be overwritten

Delete cookie

Set cookie expiration time to allready expired value

<script>
    document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 UTC";
</script>

Error handling

Errors are handled by exceptions. If error occurs outside of any try block, error in browser will occur.

Errors in browser console are unacceptable. That’s why, we are trying to catch a handle theese errors.

Exceptions

try {
    Block of code to try
}
catch(err) {
    Block of code to handle errors
} 
finally {
    Block of code to be executed regardless of the try / catch result
}

JavaScript throws standard error like "function f not defined". You can also throw your own error.

throw "my_error";   // throw a string 
throw 500;          // throw a number

Performance

Reduce DOM Size

Keep the number of elements in the HTML DOM small.

This will always improve page loading, and speed up rendering (page display), especially on smaller devices.

Every attempt to search the DOM (like getElementsByTagName) will benefit from a smaller DOM.

    <div>
        <div>
            <div>
                <span>Content</span>
            </div>
        </div>
    </div>

Performance

Avoid Unnecessary Variables

Don't create new variables if you don't plan to save values.

Often you can replace code like this: 

var fullName = firstName + " " + lastName;
document.getElementById("demo").innerHTML = fullName;

With this: 

document.getElementById("demo").innerHTML = firstName + " " + lastName

Performance

Delay JavaScript Loading

It is recommended, to include JavaScript files at the end of html document. Browser will start loading JS after page is loaded.

It’s common, to start JS functions after whole page was loaded: 

<script>
window.onload = downScripts;

function downScripts() {
    var element = document.createElement("script");
    element.src = "myScript.js";
    document.body.appendChild(element);
}
</script>
